### 介绍

本项目是实现一个简单的聊天室，聊天室分为服务端和客户端。采用C/S模型，使用TCP连接。 相关技术点:

1. 支持多个用户接入，实现多人聊天室。
2. 使用epoll机制实现并发，增加效率。
3. 使用fork创建两个进程，一个为父进程负责写入，一个为子进程负责读取。
4. 将聊天信息写到管道（pipe），并由父进程将信息写入管道，子进程将信息从管道读端读取。
5. 使用epoll机制接受服务端发来的信息，并广播信息给其他客户端。
6. 保存最近的100条记录。
